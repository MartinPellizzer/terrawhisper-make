
Below is a **reusable Koray-style entity template** for every page like:

```
/actions/anti-inflammatory.html
/actions/antimicrobial.html
/actions/antioxidant.html
/actions/adaptogenic.html
/actions/sedative.html
```

Entity example:

```
Anti-inflammatory Action
Context: Herbal Medicine
Entity Type: Pharmacological Action
```

The goal is that **each action page stands alone as a knowledge node**, meaning it defines:

* what the action is
* how it works
* what causes it
* where it acts
* what it treats
* which herbs provide it
* safety and research context

This creates a **complete Entity → Attribute → Value structure**, which is how search engines organize knowledge graphs.

# 4. Microsemantic Template (First Paragraph)

Use the **same semantic pattern** for every page.

Example:

> In herbal medicine, the **anti-inflammatory action** refers to the ability of certain medicinal plants to reduce inflammation in the body by modulating immune responses, inhibiting inflammatory enzymes, and regulating cytokine activity. This pharmacological action is produced by bioactive plant compounds such as flavonoids, terpenoids, and phenolic compounds, and plays an important role in the herbal treatment of conditions such as arthritis, digestive inflammation, and skin irritation.

This ensures the entity is immediately understood as:

```
Entity → Anti-inflammatory Action
Type → Pharmacological Action
Context → Herbal Medicine
```
